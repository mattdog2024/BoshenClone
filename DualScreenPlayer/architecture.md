# HirenderP1 克隆版技术方案与架构设计

## 1. 技术选型

为了实现一个支持 Windows 7 到 Windows 10 的双屏/多屏媒体播放软件，我们需要选择一个跨平台、支持多窗口管理、并且具备强大媒体播放能力的框架。

### 1.1 核心框架
- **Electron + React/Vue**: 适合快速开发界面，但对于多屏同步播放、硬件加速解码（特别是 4K 视频）以及精确的音视频同步控制，Electron 的性能可能无法满足专业演出需求。
- **C++ + Qt**: 性能最好，支持多屏管理，但开发周期较长。
- **Python + PyQt5/PySide2 + VLC/GStreamer**: 这是一个折中的方案。Python 开发效率高，PyQt5 提供了强大的多窗口和界面开发能力，而 VLC（通过 `python-vlc` 绑定）或 GStreamer 提供了底层强大的音视频解码能力，支持几乎所有格式，并且支持硬件加速。

**最终决定**：采用 **Python + PyQt5 + python-vlc**。
- **Python**: 开发速度快，易于维护。
- **PyQt5**: 完美支持 Windows 7-10，提供丰富的 UI 组件，支持多显示器检测和多窗口管理。
- **python-vlc**: 基于 VLC 播放器内核，支持 99% 的音视频格式，支持硬件解码，性能优异。

### 1.2 关键技术点
- **多屏管理**: 使用 `QDesktopWidget` 或 `QScreen` 获取系统显示器列表，将主控制窗口显示在主屏幕，将播放窗口全屏显示在扩展屏幕。
- **媒体播放**: 使用 `vlc.MediaPlayer` 实例嵌入到 PyQt 的 `QFrame` 中。
- **双路播放**: 需要两个 VLC 实例，一个用于主输出（大屏），一个用于预览（控制台）。
- **无缝切换/转场**: VLC 本身不支持复杂的视频转场（如淡入淡出）。为了实现淡入淡出，可以采用两个播放器实例交替播放，并通过调整窗口透明度（Opacity）或音频音量来实现淡入淡出效果。
- **数据持久化**: 使用 JSON 或 SQLite 存储播放列表和设置。

## 2. 架构设计

### 2.1 模块划分
1. **UI 模块 (ui/)**:
   - `main_window.py`: 主控制台界面（左侧列表、中间素材、右侧预览）。
   - `output_window.py`: 大屏输出窗口（全屏、无边框）。
   - `settings_dialog.py`: 设置界面。
2. **核心逻辑模块 (core/)**:
   - `player_manager.py`: 封装 VLC 播放器逻辑，管理主输出和预览两个播放器实例。
   - `playlist_manager.py`: 管理播放列表、素材的增删改查。
   - `screen_manager.py`: 管理多显示器，处理窗口在不同屏幕间的移动和全屏。
3. **数据模型 (models/)**:
   - `media_item.py`: 媒体素材类（路径、类型、时长、缩略图等）。
   - `playlist.py`: 播放列表类。
4. **工具模块 (utils/)**:
   - `thumbnail_generator.py`: 生成视频/图片缩略图。
   - `config.py`: 全局配置管理。

### 2.2 核心流程
- **启动**: 初始化配置 -> 检测显示器 -> 创建主窗口和输出窗口 -> 输出窗口移动到扩展屏并全屏 -> 加载上次的播放列表。
- **添加素材**: 用户拖拽或选择文件 -> 解析文件信息（时长、分辨率） -> 生成缩略图 -> 添加到播放列表 UI。
- **播放**: 用户双击素材 -> `player_manager` 停止当前播放 -> 加载新媒体 -> 主输出窗口和预览窗口同时开始播放。

## 3. 开发计划 (MVP 阶段)

为了快速交付一个可用的版本，我们将分阶段进行开发：

### 阶段 1：基础框架与单屏播放
- 搭建 PyQt5 项目结构。
- 实现主界面布局（列表区、控制区）。
- 集成 python-vlc，实现视频/图片的加载和播放。

### 阶段 2：双屏管理与双路播放
- 实现多显示器检测。
- 创建独立的输出窗口，并能将其全屏显示在第二屏幕。
- 实现主控制台预览和第二屏幕同步播放。

### 阶段 3：播放列表与控制
- 实现播放列表的添加、删除、排序。
- 实现播放控制（播放、暂停、进度条、音量）。
- 实现简单的播放完成动作（播放下一个、循环）。

### 阶段 4：打包与测试
- 使用 PyInstaller 将 Python 代码打包为 Windows 可执行文件 (.exe)。
- 在 Windows 环境下进行测试。
